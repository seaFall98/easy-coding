# Spec Template

## Goal

One sentence on what this feature enables.

## Context

Why this feature is needed now, and what existing behavior it changes or extends.

## Scope

What is in scope for this spec.

## Non-goals

What this spec intentionally does not solve.

## Confirmed Decisions

Decisions confirmed in grill or by accepted research.

## Approaches Considered

Only include this section when there was a real choice.

- Option A
- Option B
- Recommended option and why

## User-Facing Behavior

Describe the observable behavior in terms a reviewer can verify.

## Edge Cases

List the important failure modes, weird inputs, or boundary conditions.

## Acceptance Criteria

Observable checks that prove the feature works.

## Risks and Follow-ups

Known trade-offs, deferred work, or future expansion points.

## Open Questions

Anything that still blocks planning.

## Self-Review Checklist

- No placeholders
- No contradictions
- Scope is narrow enough for one plan
- Terms match the project glossary
- Every major behavior has a source
- Acceptance criteria are observable
